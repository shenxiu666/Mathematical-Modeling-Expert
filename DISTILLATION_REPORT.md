## 一、蒸馏概要

本报告记录了从顶尖数学建模专家的认知模式出发，蒸馏并构建一个可在 Claude Code 中使用的 **数学建模专家 Skill** 的完整过程。Skill 融合了陶哲轩的降维简化思维与运筹学专家的实战落地思维，具备从现实问题抽象到数值求解的完整建模能力。

## 二、提炼的 3 个核心建模思维特征

### 特征 1：逐层降维与假设显式化（奥卡姆剃刀原则）

**来源**：陶哲轩在解决复杂数学问题时，习惯先构造极度简化的"玩具模型"验证核心逻辑，再逐步引入复杂性。

**在本 Skill 中的体现**：
- 强制要求建模者先列出一个 **Toy Model**（仅保留 2-4 个核心变量）
- 每个假设必须**编号、可验证**，并说明合理性与脆弱性
- 从理想情况出发，逐步松弛假设（如从欧氏距离 → 曼哈顿距离 → 时变拥堵路网）

**效果**：避免"一上来就堆砌复杂模型"的低效建模方式，符合 CUMCM/MCM 评审对"假设合理性"的强调。

### 特征 2：守恒律与量纲一致性检验（物理思维嵌入）

**来源**：应用数学中，任何描述现实系统的方程必须满足基本的守恒律（质量、能量、资金流等）。

**在本 Skill 中的体现**：
- Phase 2（模型构建）中强制执行**量纲一致性检查**
- 提供检查清单：流入 = 流出 + 累积、参数物理意义验证、极端条件行为检验
- 在 Few-Shot 示例中展示具体的量纲检查过程

**效果**：有效过滤掉"量纲不一致""违背守恒律"的错误模型，提高模型的可信度。

### 特征 3：算法可实现性约束（实战导向）

**来源**：运筹学专家的核心素养——模型不仅要在数学上优美，更要在计算上可行。

**在本 Skill 中的体现**：
- 强制要求在提出模型后给出**时间复杂度估计**
- NP-Hard 问题必须提供**启发式求解框架**（贪心、VNS、遗传算法等）
- 配备实际的 **Python 工具链**（`optimization_solver.py` + `sensitivity_analyzer.py`）用于验证

**效果**：确保输出的模型是"可落地的"，满足工程和竞赛的双重标准。

### 5.3 局限性

1. **专业仿真软件替代**：本 Skill 的工具链适用于中小规模问题。对于超大规模问题（百万级整数变量）或复杂三维流体仿真，需要 Gurobi/COMSOL/ANSYS 等专业软件。
2. **领域知识深度**：Skill 提供的是建模方法论框架，对于高度专业化的领域（如量子力学、分子动力学），仍需结合领域专家知识。
3. **工具链覆盖范围**：当前工具链覆盖 LP/IP/NLP 求解和单因子灵敏度分析。尚未包含微分方程数值求解（可使用 `scipy.integrate` 直接调用）和统计建模工具（可结合 `statsmodels`）。

### 5.4 综合评分

**总体评估**：该 Skill 具备**参加并胜任** CUMCM 和 MCM/ICM 竞赛的核心能力，尤其在优化调度、网络分析、评价决策类题型上表现突出。对于物理和工程建模题型，Skill 提供的思维框架有效，但建议用户结合领域专业工具进行最终求解。

**自评分数**：★★★★☆ (4.2/5.0)
